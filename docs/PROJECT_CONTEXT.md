Kalitka

Описание проекта

Kalitka — это современный VPN-сервис, построенный вокруг максимальной простоты использования.

Главная цель проекта — чтобы пользователь смог получить рабочее VPN-подключение за минимальное количество действий.

Проект разрабатывается как Progressive Web App (PWA), которое выглядит и ощущается как полноценное мобильное приложение.

⸻

Основная идея

Пользователь не должен видеть технические детали.

Он не должен знать что такое:

* VLESS
* Reality
* XHTTP
* Hysteria 2
* UUID
* Subscription URL

Все технические процессы должны происходить автоматически.

Интерфейс должен быть максимально простым.

⸻

Целевая аудитория

Обычные пользователи.

Даже человек без технических знаний должен суметь подключить VPN.

⸻

Основные принципы проекта

* Максимально простой интерфейс.
* Минимальное количество действий.
* Современный минималистичный дизайн.
* Высокая скорость работы.
* Адаптация под мобильные устройства.
* Код должен быть масштабируемым.
* Архитектура должна быть модульной.

VPN

Используемые протоколы

* VLESS Reality
* XHTTP
* Hysteria 2

На старте используется один сервер.

Локация:

Germany

Архитектура должна позволять легко добавить новые серверы в будущем.

⸻

Бесплатный доступ

Каждый пользователь может получить бесплатный VPN один раз.

Бесплатный тариф:

* 1 ГБ

После использования бесплатного трафика пользователь может приобрести подписку.

⸻

Авторизация

На первом этапе регистрация максимально упрощена.

Пользователь сначала получает VPN.

После этого ему предлагается сохранить доступ к аккаунту.

Поддерживаемые способы:

* Apple
* Google
* Email (magic link)

Использование паролей не планируется.

⸻

PWA

Проект должен работать как полноценное приложение.

Поддерживается:

* установка на главный экран
* адаптация под iOS
* адаптация под Android
* Push-уведомления (в будущем)

⸻

VPN-клиент

Основной клиент:

Happ

Архитектура должна позволять заменить VPN-клиент без изменения основной логики проекта.

Без длинных описаний.

Без сложной регистрации.

Интерфейс должен быть максимально простым.

⸻

Поддержка

В проект встроен собственный чат поддержки.

Пользователь пишет сообщения прямо на сайте.

Все сообщения автоматически пересылаются администратору через Telegram-бота.

Ответ администратора автоматически появляется в чате пользователя.

Telegram используется только как административный интерфейс.

⸻

Оплата

На первом этапе используется ручное подтверждение оплаты.

Сценарий:

1. Пользователь запрашивает продление.
2. Администратор отправляет QR-код СБП.
3. Пользователь оплачивает.
4. Администратор подтверждает оплату.
5. Система автоматически продлевает подписку через API 3x-ui.

Архитектура должна позволять позже заменить ручную оплату на полностью автоматическую без изменения основной логики проекта.

⸻

Интеграция с 3x-ui

Все операции с VPN выполняются исключительно через официальный API 3x-ui.

Не допускается ручное управление пользователями через код проекта.

⸻

Принципы разработки

Перед выполнением любой задачи разработчик обязан ознакомиться с данным документом.

При возникновении неоднозначности необходимо придерживаться следующих принципов:

* простота важнее количества функций;
* читаемость важнее сложных решений;
* модульность важнее монолитной архитектуры;
* безопасность важнее скорости разработки;
* пользовательский опыт всегда имеет наивысший приоритет.

Новые функции должны органично вписываться в существующую архитектуру и не усложнять интерфейс.

Проект создаётся как долгосрочный коммерческий продукт, рассчитанный на дальнейшее масштабирование.
