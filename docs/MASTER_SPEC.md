# Master Spec

## Contents

1. [Vision](#vision)
Цель проекта

Kalitka — это современный VPN-сервис, основная задача которого сделать получение VPN максимально простым.

Пользователь не должен разбираться в протоколах, настройках и технических терминах.

Он должен получить рабочий VPN за несколько нажатий.

Проект разрабатывается как коммерческий продукт с возможностью масштабирования.

⸻

Философия продукта

Главный принцип:

Максимальная простота.

Каждая функция должна отвечать на вопрос:

Делает ли она сервис проще для пользователя?

Если ответ “нет”, функция не должна быть реализована.

⸻

Основные ценности

* Простота.
* Скорость.
* Надёжность.
* Минимализм.
* Современный дизайн.
* Масштабируемость.

⸻

Целевая аудитория

Обычные пользователи.

Не предполагается наличие технических знаний.

⸻

Основной сценарий

Пользователь:

1. Сканирует QR-код.
2. Переходит на сайт.
3. Получает бесплатный VPN.
4. Устанавливает Happ.
5. Подключается.
6. Пользуется VPN.
7. При необходимости продлевает доступ.

2. [Product Goals](#product-goals)
Проект должен решить следующие задачи.

Для пользователя

Максимально быстро получить VPN.

Не проходить сложную регистрацию.

Не изучать инструкции.

Не копировать длинные ссылки.

Не выполнять ручные настройки.

⸻

Для администратора

Минимум ручной работы.

Все действия должны выполняться максимально быстро.

Основной рабочий инструмент администратора — Telegram.

Административная панель используется только для просмотра статистики и управления системой.

⸻

Основные принципы UX

Минимальное количество экранов.

Минимальное количество кнопок.

Минимальное количество полей ввода.

Каждое действие должно быть понятным без инструкции.

⸻

MVP

Первая версия должна содержать:

* Главную страницу.
* Получение бесплатного доступа (1 ГБ).
* Интеграцию с API 3x-ui.
* Личный кабинет.
* Встроенную поддержку.
* Telegram-бот администратора.
* Ручное продление подписки.
* PWA.

Любая дополнительная функциональность реализуется только после завершения MVP.
3. [User Journey](#user-journey)
Первый запуск

Пользователь сканирует QR-код.

↓

Открывается

https://kalitka.jesarion.com

↓

Главная страница.

↓

Кнопка

“Получить 1 ГБ бесплатно”

↓

Автоматическое создание пользователя.

↓

Создание клиента в 3x-ui.

↓

Получение VPN.

↓

Установка Happ.

↓

Подключение.

↓

Главный экран личного кабинета.

⸻

Главный принцип разработки

Во время разработки всегда задавать вопрос:

“Можно ли убрать ещё один шаг?”

Если да — убрать.

Если нет — оставить.

Kalitka должна ощущаться как полноценное мобильное приложение, а не как обычный веб-сайт.
